# ReplyKit Pro

A Next.js cold outreach template library built for founders, freelancers, and growth teams.

> NOTE: Payments are currently disabled in this repo (Stripe integration removed).
> If you want to accept payments later, consider providers like PayPal, Paddle,
> Gumroad, Razorpay (India), or Paystack. See the section below for suggested
> options and a quick recommendation for international students.

## What to set up

### 1. Loops newsletter form ID

1. Go to https://loops.so and sign up or log in.
2. Create a new newsletter form or widget.
3. Copy the form ID from the embed/settings screen.
4. Add it to `.env.local` as:

```env
NEXT_PUBLIC_LOOPS_FORM_ID=your_loops_form_id
```

### 2. Supabase project

1. Go to https://supabase.com and create an account.
2. Create a new project.
3. Open the project dashboard and go to `Settings > API`.
4. Copy:
   - `Project URL`
   - `anon key`
5. Add them to `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-public-anon-key
```

### 3. Create the submission table

In Supabase SQL editor, run:

```sql
create table template_submissions (
  id bigint generated by default as identity primary key,
  name text,
  email text,
  title text,
  category text,
  channel text,
  industry text,
  description text,
  body text,
  created_at timestamp with time zone default timezone('utc', now())
);
```

### 4. Stripe and Supabase admin setup

If you want paid plans and an admin dashboard, also create these tables:

```sql
create table pro_users (
  id bigint generated by default as identity primary key,
  email text,
  stripe_customer_id text,
  stripe_subscription_id text,
  subscription_status text,
  current_period_end timestamp with time zone,
  created_at timestamp with time zone default timezone('utc', now())
);

create table events (
  id bigint generated by default as identity primary key,
  event_type text,
  metadata jsonb,
  created_at timestamp with time zone default timezone('utc', now())
);
```

### 5. Run locally

```bash
npm install
cp .env.example .env.local
# edit .env.local with your keys
npm run dev
```

### Notes

- `components/NewsletterSignup.tsx` uses `NEXT_PUBLIC_LOOPS_FORM_ID`.
- `app/api/submit/route.ts` saves submit form data to Supabase.
- `app/api/checkout/route.ts` uses Stripe price IDs and `NEXT_PUBLIC_URL`.
- `app/api/personalize/route.ts` uses `OPENROUTER_API_KEY`.
- `app/admin/page.tsx` requires `SUPABASE_SERVICE_ROLE_KEY`.
